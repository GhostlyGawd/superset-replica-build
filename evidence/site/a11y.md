# apps/site — accessibility audit (axe-core)

**Surface:** Grove launch site (`apps/site`, ADR-0021) — the prerendered static cockpit page.
**Tool:** axe-core 4.12.1 (MIT), injected by Playwright over the served `dist` (`vite preview`).
**Spec:** `apps/site/e2e/a11y.spec.ts` — a HARD GATE of **0 critical / 0 serious**; lesser impacts
are recorded, not failed. Raw machine output: `evidence/site/a11y-results.json`.
**Date:** 2026-06-16. Generated on Windows; the same suite runs in CI (`site` job, ubuntu) on every PR.

## Rule set

`runOnly` tags: `wcag2a`, `wcag2aa`, `wcag21a`, `wcag21aa`, `best-practice` — i.e. the full
WCAG 2.0/2.1 A + AA rule set plus axe's best-practice rules (the same tag set the desktop audit
in `apps/desktop/e2e/_a11y.spec.ts` uses).

## The one remaining finding (moderate — below the gate)

- `page-has-heading-one` (impact: **moderate**) ×1 on the page-at-rest surfaces.
  The cockpit concept is deliberately **not** a centred-hero marketing page: the top of the
  document is the product's real pinned chrome and the single 30px line lives inline inside it as
  an `<h2>` (DESIGN.md §type: "the 30px `3xl` appears exactly once" + anti-generic guardrail #1
  "No centered hero — structurally impossible"). axe's best-practice rule prefers a single `<h1>`;
  this is an intentional structural choice, not a barrier (it is moderate, not serious/critical,
  and every section has a programmatic heading + the document has a real `<title>`). Left as-is to
  honour the design law; recorded here for transparency.

## A serious finding that WAS fixed during this work

The first phone run surfaced **`scrollable-region-focusable` (serious) ×3** — the roster table
wrapper, the worktree-layout `CodeBlock`, and the diff body became horizontally/vertically
scrollable at the phone width but were not keyboard-focusable, so a keyboard-only user could not
scroll them. Fixed by making each scroll container keyboard-focusable (`tabindex=0` + an
`aria-label` + a focus ring) in `@swarm/ui` (`DiffView` body, `CodeBlock` `<pre>`, `TerminalFrame`
body) and in the site sections (`ColdOpen` roster wrapper, `Harvest` split-diff columns). Re-run:
**0 serious.**
